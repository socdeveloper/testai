Database Mastery - Lesson 1: PostgreSQL vs MongoDB

Learn when and how to use PostgreSQL and MongoDB effectively

PostgreSQL vs MongoDB

Understanding the differences between relational (PostgreSQL) and document (MongoDB) databases is crucial for backend development.

PostgreSQL: Relational Database

PostgreSQL is a powerful, open-source object-relational database system known for its reliability, feature robustness, and performance.

When to use PostgreSQL:

  • When you need ACID compliance
  • Complex queries with joins
  • Structured data with defined schemas
  • Strong consistency requirements
  • Financial applications
-- Creating a users table CREATE TABLE users ( id SERIAL PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(255) UNIQUE NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Inserting data INSERT INTO users (name, email) VALUES ('John Doe', 'john@example.com'); -- Querying data SELECT * FROM users WHERE email = 'john@example.com';

MongoDB: Document Database

MongoDB is a NoSQL database that stores data in flexible, JSON-like documents with dynamic schemas.

When to use MongoDB:

  • Unstructured or semi-structured data
  • Horizontal scaling needs
  • Agile development with changing requirements
  • Content management systems
  • Real-time analytics
// Creating a users collection db.users.insertOne({ name: "John Doe", email: "john@example.com", createdAt: new Date() }); // Querying data db.users.find({email: "john@example.com"}); // Aggregation pipeline db.orders.aggregate([ {$match: {status: "completed"}}, {$group: {_id: "$customerId", total: {$sum: "$amount"}}} ]);

Indexing Strategies

Indexes are crucial for performance in both PostgreSQL and MongoDB, but they work differently.

PostgreSQL Indexes

-- B-tree index (default) CREATE INDEX idx_users_email ON users(email); -- Partial index CREATE INDEX idx_active_users ON users(id) WHERE active = true; -- Composite index CREATE INDEX idx_users_name_age ON users(name, age);

MongoDB Indexes

// Single field index db.users.createIndex({email: 1}) // Compound index db.users.createIndex({name: 1, age: 1}) // Text index db.users.createIndex({name: "text", bio: "text"})
