Backend Developer Interview Preparation

This section covers common interview questions and scenarios you'll encounter when interviewing for backend positions.

System Design Questions

Design a URL Shortening Service

Design a system like Bitly that can handle millions of requests per day.

Considerations:

  • How to generate unique short codes
  • How to handle high QPS (queries per second)
  • How to ensure data consistency
  • How to scale the system
  • How to handle analytics

Approach:

1. API Design: POST /shorten, GET /{code}

2. Database Choice: PostgreSQL for URL mapping, MongoDB for analytics

3. Architecture: Go microservices with Redis for caching

4. Scalability: Use consistent hashing for distributed storage

5. Performance: CDN for redirects, Reindexer for fast lookups

Design a Real-Time Notification System

Design a system that can send notifications to millions of users with sub-second latency.

Approach:

1. Architecture: Event-driven with Redpanda for message streaming

2. Services: Go microservices for different notification types

3. Data Storage: PostgreSQL for user preferences, MongoDB for history

4. Delivery: WebSocket connections or push notifications

5. Scalability: Horizontal partitioning by user ID

Technical Questions

Explain the difference between horizontal and vertical scaling. When would you use each?

Vertical Scaling (Scale-up): Increasing the capacity of existing servers by adding more CPU, RAM, or storage. Pros: Simple to implement, no changes to application needed. Cons: Hardware limits, single point of failure, expensive.

Horizontal Scaling (Scale-out): Adding more servers to distribute the load. Pros: Better fault tolerance, virtually unlimited scaling, cost-effective. Cons: More complex, requires application changes.

When to use: Vertical scaling for small applications or databases that don't support sharding. Horizontal scaling for high-traffic applications, microservices, and distributed systems.

How would you design an API in Go that can handle 10,000 requests per second?

1. Framework: Use a high-performance framework like Gin or Echo

2. Concurrency: Leverage goroutines for concurrent request handling

3. Caching: Implement Redis for frequently accessed data

4. Database: Use connection pooling, proper indexing, and read replicas

5. Load Balancing: Distribute requests across multiple instances

6. Monitoring: Implement Prometheus metrics for performance tracking

How do you ensure data consistency in a distributed system?

1. Two-Phase Commit (2PC): For strong consistency across distributed transactions

2. Eventual Consistency: Using event sourcing and CQRS patterns

3. Saga Pattern: For long-running transactions with compensating actions

4. Idempotency: Ensuring operations can be safely repeated

5. Message Queues: Using Redpanda with exactly-once semantics

6. Consensus Algorithms: Raft or Paxos for distributed coordination
